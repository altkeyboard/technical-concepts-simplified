# What is Git?

## Plain-English definition
Git is a version control system that records changes to files over time so you can review history, collaborate safely, and restore earlier versions if needed.

## Why teams use it
- **Traceability:** See who changed what and why.
- **Collaboration:** Multiple people can work in parallel without overwriting each other.
- **Change control:** Edits can be reviewed before they become “official.”

## Key terms
- **Commit:** A saved snapshot of changes with a message describing what changed.
- **Branch:** A parallel copy of work used for drafts or new changes.
- **Merge:** Combining approved changes back into the main version.

## When it’s especially useful
Git shines for text-based content like documentation (Markdown), configuration, and code because it can compare changes line-by-line.
