## Plain-English definition
CI/CD is an automated way to build, test, and deliver updates.

- **CI (Continuous Integration):** automatically checks changes when they’re submitted
- **CD (Continuous Delivery/Deployment):** automatically packages and releases changes

## Why teams use it
- Faster releases
- Fewer errors
- Repeatable, auditable processes

## How docs fit in
Documentation updates can be reviewed in pull requests and automatically published to a website when merged.
