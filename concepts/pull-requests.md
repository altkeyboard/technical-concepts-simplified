# What is a Pull Request?

## Plain-English definition
A pull request (PR) is a request to review and merge changes from a branch into the main version of a project.

## Why it matters
PRs create a clear review process:
- Changes are visible (diffs)
- Feedback can be captured
- Approvals are recorded
- Merges create a traceable history

## Typical PR flow
1. Create a branch
2. Make changes and commit them
3. Open a pull request
4. Review and revise
5. Merge into main
